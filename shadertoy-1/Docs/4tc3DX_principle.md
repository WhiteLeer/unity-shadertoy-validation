# 4tc3DX (Glorious Line Algorithm) 原理解析

## 1. 效果类型
- 单 pass、纯程序化（无贴图输入）的 2D 几何绘制 shader。
- 核心是用 **SDF（有符号距离场）** 统一描述线段、圆角矩形、描边、虚线。

## 2. 主体流程
1. 将屏幕坐标映射到局部 `uv`，并按宽高比修正，保证形状不拉伸。
2. 构造若干关键点（pA/pB/pC/pD），并用时间驱动旋转，让图形动态变化。
3. 对每个目标形状计算距离场：
   - 线方向距离（dpx）
   - 法线方向距离（dpy）
   - 端点圆角处理
4. 把距离转换为遮罩（填充或描边），通过乘法叠加到最终颜色。
5. 用周期函数做虚线节奏和背景网格。
6. 最后做轻量色调修正输出。

## 3. 关键函数职责
- `LineDistField`：统一计算“到线/圆角端点形状边界”的距离。
- `FillLinePix`：像素单位的填充线，线宽随分辨率保持稳定。
- `DrawOutlinePix`：像素单位描边。
- `FillLine / FillLineDash`：UV 单位线条（含虚线版本）。
- `DrawOutline`：UV 单位描边。

## 4. 抗锯齿机制
- 依赖 `dFdy`（以及局部导数尺度）把距离场转换为平滑过渡边缘。
- 本质是基于屏幕导数估计“一个像素对应的距离范围”，避免硬阈值锯齿。

## 5. 为什么能“一套逻辑画很多形状”
- SDF 的好处是：只要有“距离到边界”的定义，填充、描边、圆角、虚线都能复用同一套后处理。
- 这比传统分支式图元绘制更统一，便于动画和参数化。

## 6. Unity 复现对应
- `iResolution -> _ScreenParams.xy`
- `iTime -> _Time.y`
- 保持 full-screen quad + 正交相机铺满（按相机可视宽高动态缩放 quad）。
- 分辨率由 `shadertoy-1-capture.resolution.json` 自动驱动，保证构图与 Shadertoy 一致。
