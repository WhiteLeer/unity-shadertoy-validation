# NflSD8 (Unimagined) 原理解析

## 1. 效果类型
- 这是一个 **单 pass、无贴图输入** 的全屏体渲染风格效果。
- 主要视觉来自体积雾/能量云式的 **raymarch 累积发光**。

## 2. 主体流程
1. 从屏幕坐标构造视线方向。
2. 沿视线做固定步数（50 次）march。
3. 每一步把采样点做时变旋转、平移和周期平铺（mod）。
4. 用一个混合距离函数估计局部“密度”v（球壳感 + 线结构感）。
5. 用 v 控制步进长度（稀疏处快走，稠密处慢走）。
6. 把 `exp(sin(...)) / v` 作为能量项累加到颜色。
7. 循环结束后做 `tanh` 色调映射压高亮。

## 3. 关键公式直觉
- `p = t * normalize(...)`：把当前行进距离 t 投到视线上。
- `p.xy *= rotation(t)`：采样空间随 t 旋转，形成扭曲流动感。
- `p = mod(p, 4.0) - 2.0`：把空间折叠为重复单元，产生无限重复结构。
- `v = lerp(abs(length(p)-1), length(p.xz), oscillation) + 0.01`：
  在“球壳 SDF”和“柱线感距离”之间随 t 插值，结构会随距离变化。
- `t += v * 0.3`：自适应步进，兼顾速度与细节。
- `O += exp(sin(t + phase))/v`：类似体发光积分，v 越小（越靠近结构）贡献越大。

## 4. 为什么看起来像体积效果
- 不是画单个表面，而是沿每条像素射线多次采样并累计颜色。
- 这种“沿深度积分”天然会有云雾、能量、体腔感。

## 5. 性能与质量点
- 成本主要在每像素 50 次循环 + 三角函数/exp。
- 迭代次数越高细节越稳，但成本线性上升。
- `tanh` 压亮可避免过曝，但会降低对比上限。

## 6. Unity 复现对应
- `iResolution -> _ScreenParams.xy`
- `iTime -> _Time.y`
- 保持 full-screen quad + 正交相机铺满
- 分辨率由 `shadertoy-0-capture.resolution.json` 驱动，避免构图偏差
